Trace Extension Page

When checking "Trace Extension to Next Output", we can override the "Evaluation Steps" limit defined in the Common Criteria Tab. This allows to extend a given symbolic execution configuration for certain purposes. We can add a fixed number of steps or allow an infinite number of steps by entering "-1".

As we can guess, this file records events as they unfold during the symbolic execution. Those events can be (among others) :

  • when a machine starts ("run" keyword on a statemachine, a state ...)
  • when a transition is triggered ("fire" keyword)
  • when an input communication port is used ("INPUT" keyword )
  • when an output communication port is used ("OUTPUT" keyword )
  • ...

When several branches are explored, the diverse possibilities are recorded under different sections named "TRACE NUMBER X".

We can lighten this report or flesh it out using the following options :

  • in "Observable Input/Output Selection" :
    • "None" will retain no communication at all in the report
    • "Input/Output" will retain all "INPUT" and "OUTPUT" external events
    • "Input" will retain only external "INPUT" events
    • "Output" will retain only external "OUTPUT" events
    • "All" will retain all communication events (both "INPUT" and "OUTPUT", both external (to the environment) and internal (between machines...))
    • by selecting "Details" you can filter out which communication events to track by giving out their names
  • in "Observable Transition Selection" :
    • "None" will retain no transitions at all in the report
    • "All" will retain all transition events
    • by selecting "Details" you can filter out which transitional events to track by giving out their names

By checking "Redundance Elimination", you allow Diversity to prune some of the "TRACE NUMBER X" sections if several of them have identical contents.
