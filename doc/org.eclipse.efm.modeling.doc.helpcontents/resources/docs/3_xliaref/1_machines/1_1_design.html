Model-instance paradigm

Formalism

XLIA supports the "model-instance" paradigm. Thus, a machine can be defined to be a :

  • "model", which is destined to be instantiated, once or several times, in the sense of object-oriented programming of this concept
  • "instance", that can be distinguished through its initialization, total or partial, of the parameters of its model when they exist, similar to the way to handle class constructors in the Object paradigm
  • "prototype", a syntactic shortcut (or macro) to, in one fell swoop, define a model and declare its first instance, its parameters if any could be initialized during declaration

The default mode, in a syntactic economy perspective, is that each machine is a "prototype".

Purpose of the model-instance paradigm

So, if the two formalisms can lead to the same thing, what's the point in creating a model and an instance ?

In facts, one of the forces of the "model-instance paradigm" is that we can , from a single model, create different instance, and not just equivalent instances with different names.
